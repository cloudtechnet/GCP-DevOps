## Audit Logs & IAM Monitoring in GCP 


# 1️⃣ What are Audit Logs in GCP?

**Cloud Audit Logs** record all **activities performed on Google Cloud resources**.

They help administrators track:

* Who accessed a resource
* What action was performed
* When the action happened
* From where the request came

Simple definition:

> **Audit Logs record every administrative and API activity in Google Cloud for security and monitoring purposes.**

---

# 2️⃣ Types of Audit Logs in GCP

| Log Type                | Description                                                        |
| ----------------------- | ------------------------------------------------------------------ |
| **Admin Activity Logs** | Records administrative actions like creating or deleting resources |
| **Data Access Logs**    | Records access to data such as reading objects in storage          |
| **System Event Logs**   | Events triggered by Google systems                                 |
| **Policy Denied Logs**  | Logs when IAM denies access                                        |

Example event:

```
User: devops@company.com
Action: compute.instances.delete
Resource: VM instance
Time: 10:30 AM
```

---

# 3️⃣ What is IAM Monitoring?

**IAM Monitoring** means observing **who is accessing resources and whether access is authorized**.

Tools used:

* **Cloud Logging**
* **Logs Explorer**
* **Cloud Monitoring**
* **Security Command Center**

IAM monitoring helps detect:

* Unauthorized access
* Suspicious activity
* Misconfigured permissions

---

# 4️⃣ Real-World Scenario

### Scenario

A company has **production VM servers**.

Security team wants to monitor:

* Who creates VMs
* Who deletes VMs
* Who changes IAM permissions

If a developer accidentally deletes a VM, the **Audit Log records the activity**.

Example log entry:

```
User: developer@company.com
Action: compute.instances.delete
Resource: prod-vm-01
Status: SUCCESS
```

This helps the security team **investigate the incident**.

# 5️⃣ Step-by-Step LAB – Audit Logs & IAM Monitoring

This lab shows how to **track IAM changes and VM creation activity**.

---

# Step 1: Create a Test VM

Go to:

```
Compute Engine → VM Instances
```

Click:

```
Create Instance
```

Example configuration:

```
Name: audit-test-vm
Region: us-central1
Machine Type: e2-micro
```

Click **Create**

Now an activity is generated in **Audit Logs**.

---

# Step 2: Open Cloud Logging

Go to:

```
Navigation Menu → Logging → Logs Explorer
```

This is where all **audit logs are stored**.

---

# Step 3: Filter Audit Logs

In **Logs Explorer**, use the query:

```
resource.type="gce_instance"
protoPayload.methodName="v1.compute.instances.insert"
```

This query shows **VM creation logs**.

You will see output like:

```
User: rajesh@company.com
Action: compute.instances.insert
Resource: audit-test-vm
Status: SUCCESS
```

---

# Step 4: Monitor IAM Changes

Now let's track **IAM permission changes**.

Go to:

```
IAM & Admin → IAM
```

Add a new role to a user.

Example:

```
User: devops@company.com
Role: Viewer
```

Click **Save**.

---

# Step 5: Check IAM Audit Logs

Open **Logs Explorer** again.

Use this query:

```
protoPayload.methodName="SetIamPolicy"
```

You will see log entries like:

```
User: admin@company.com
Action: SetIamPolicy
Resource: project
Role Added: Viewer
```

This shows **who changed IAM permissions**.

# 7️⃣ Monitoring Suspicious Activities

Security teams monitor logs for:

| Suspicious Activity | Example               |
| ------------------- | --------------------- |
| Unauthorized access | Unknown user login    |
| VM deletion         | Production VM deleted |
| IAM role changes    | Admin role assigned   |

Example query to detect VM deletion:

```
protoPayload.methodName="v1.compute.instances.delete"
```

> Cloud Audit Logs in GCP record all administrative and API activities performed on cloud resources. IAM monitoring uses these logs to track access, permission changes, and resource activities.
