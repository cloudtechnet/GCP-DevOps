## GCP IAM ?

**GCP IAM (Identity and Access Management)** is a security service in **Google Cloud Platform (GCP)** that controls **who can access cloud resources and what actions they can perform**.

In simple words:
👉 **IAM decides who can do what on which resource in Google Cloud.**

Example:

* A **DevOps engineer** can create and manage Virtual Machines.
* A **developer** can only view the application logs.
* A **tester** can only access the testing environment.

IAM helps organizations **secure their cloud infrastructure** by giving the **right access to the right user**.

---

### Basic Idea of IAM

IAM works with three simple things:

**1️⃣ Who (Identity / Principal)**
The person or service requesting access.

Examples:

* User ([rajesh@gmail.com](mailto:rajesh@gmail.com))
* Google Group
* Service Account
* Domain

**2️⃣ What Access (Role)**
A role is a group of permissions.

Example roles:

* Viewer → only read access
* Editor → modify resources
* Admin → full control

**3️⃣ Which Resource**
The cloud resource the user wants to access.

Examples:

* Virtual Machine
* Storage Bucket
* Kubernetes Cluster
* Database

---

### Simple Example

Suppose a user **Ravi** needs to view storage buckets.

IAM configuration:

* **User:** [ravi@gmail.com](mailto:ravi@gmail.com)
* **Role:** Storage Viewer
* **Resource:** Storage Bucket

Result:
✔ Ravi can **see the bucket**
❌ Ravi **cannot delete or modify the bucket**

---

### Why IAM is Important

IAM is used to:

* 🔐 Protect cloud resources
* 👥 Control user access
* ⚙️ Manage permissions securely
* 🏢 Support enterprise security policies

---

✅ **One-line definition (best for interviews):**

> **GCP IAM is a service that controls authentication and authorization, allowing administrators to manage who can access Google Cloud resources and what actions they can perform.**
