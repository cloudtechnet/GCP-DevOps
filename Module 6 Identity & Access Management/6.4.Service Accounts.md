## 1️⃣ What is a Service Account?

A **Service Account** in **Google Cloud Platform (GCP)** is a **special type of account used by applications, services, or automation tools to interact with GCP resources**.

Unlike normal user accounts (used by people), **service accounts are used by machines or software**.

Simple definition:

> **A Service Account is a machine identity used by applications or automation tools to access Google Cloud resources securely.**

---

## 2️⃣ Why Service Accounts are Used

In cloud environments, many tasks are automated.

Examples:

* CI/CD pipelines
* Terraform infrastructure creation
* Applications accessing databases
* Kubernetes workloads accessing cloud services

Instead of using a **human user account**, these systems use **Service Accounts**.

This improves:

* Security
* Automation
* Access control

---

## 3️⃣ Example of Service Account Format

A service account looks like this:

```
terraform-sa@project-id.iam.gserviceaccount.com
```

Components:

* **terraform-sa** → service account name
* **project-id** → GCP project
* **iam.gserviceaccount.com** → domain

---

# 4️⃣ Real-World Example (DevOps CI/CD Pipeline)

### Scenario

A company is using **Terraform to create infrastructure in GCP**.

Terraform needs permissions to:

* Create VM instances
* Create networks
* Create storage buckets

But Terraform is not a user.

So we create a **Service Account**.

---

### Step 1: Create Service Account

Example:

```
terraform-sa@myproject.iam.gserviceaccount.com
```

---

### Step 2: Assign Role

Give required IAM role.

Example roles:

```
Compute Admin
Storage Admin
```

Now the service account can:

* Create VM instances
* Create storage buckets
* Manage infrastructure

---

### Step 3: Terraform Uses Service Account

Terraform authenticates using:

* **Service account key**
* **Workload identity**
* **Application default credentials**

Terraform command example:

```
terraform apply
```

Terraform uses the **service account identity** to create resources in GCP.

---

# 5️⃣ Another Real-World Example (Kubernetes / GKE)

### Scenario

An application running in **GKE (Kubernetes)** needs to:

* Read files from **Cloud Storage**

Solution:

Create a **service account**.

Example:

```
app-storage-sa@project-id.iam.gserviceaccount.com
```

Assign role:

```
Storage Object Viewer
```

Now the application can:

✔ Read files from storage bucket
❌ Cannot delete bucket

---

# 6️⃣ Types of Service Accounts in GCP

### 1️⃣ User-Managed Service Accounts

Created by users for applications.

Example:

```
jenkins-sa
terraform-sa
app-service-sa
```

Used for:

* CI/CD
* Automation
* Applications

---

### 2️⃣ Google-Managed Service Accounts

Automatically created by Google services.

Example:

```
Compute Engine default service account
App Engine service account
Cloud Functions service account
```

Used internally by GCP services.

---

# 7️⃣ Security Best Practices

### ✔ Follow Least Privilege

Only give required permissions.

Example:

Instead of:

```
Compute Admin
```

Use:

```
Compute Instance Admin
```

---

### ✔ Avoid Long-Lived Keys

Prefer:

* **Workload Identity**
* **IAM roles**

Instead of service account keys.

---

### ✔ Use Different Service Accounts

Example:

| Service     | Service Account |
| ----------- | --------------- |
| Terraform   | terraform-sa    |
| Jenkins     | jenkins-sa      |
| Application | app-service-sa  |

---

# 8️⃣ Simple Interview Answer

> A Service Account in Google Cloud is a special account used by applications, virtual machines, or automation tools to access GCP resources. Instead of using user credentials, applications authenticate using service accounts and are granted IAM roles to perform specific tasks securely.
