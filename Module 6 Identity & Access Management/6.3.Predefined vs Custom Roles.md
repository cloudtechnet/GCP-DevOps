## Predefined Roles vs Custom Roles (GCP IAM)

In **Google Cloud IAM**, roles define **what actions a user or service account can perform** on cloud resources.
Two important types of roles are **Predefined Roles** and **Custom Roles**.

---

## 1️⃣ Predefined Roles

**Predefined roles** are roles **created and maintained by Google Cloud**.
They contain a **predefined set of permissions for specific services**.

### Key Characteristics

* Created by **Google**
* Designed for **specific services**
* Automatically **updated by Google**
* Easy to use

### Examples

| Predefined Role         | Description                        |
| ----------------------- | ---------------------------------- |
| Compute Admin           | Full access to manage VM instances |
| Storage Admin           | Full control of storage buckets    |
| Kubernetes Engine Admin | Manage GKE clusters                |
| Logging Viewer          | View logs only                     |

Example:

```
roles/compute.admin
```

Permissions included:

```
compute.instances.create
compute.instances.delete
compute.instances.start
compute.instances.stop
```

### Example Scenario

A **DevOps engineer** managing VM infrastructure can be assigned:

* **Role:** Compute Admin

This allows them to **create, delete, start, and stop VM instances**.

---

## 2️⃣ Custom Roles

**Custom roles** are roles **created by the organization or administrator** with **specific permissions**.

They are used when predefined roles **provide too many permissions**.

### Key Characteristics

* Created by **organization or project admin**
* Contains **selected permissions only**
* Provides **fine-grained access control**
* Helps follow **least privilege principle**

### Example

Suppose a developer only needs to **start and stop VM instances**, but **should not delete them**.

Custom role permissions:

```
compute.instances.start
compute.instances.stop
compute.instances.get
compute.instances.list
```

Example role name:

```
devopsVmOperator
```

Now the developer **cannot delete VMs**, improving security.

---

## 3️⃣ Difference Between Predefined and Custom Roles

| Feature     | Predefined Roles      | Custom Roles                |
| ----------- | --------------------- | --------------------------- |
| Created by  | Google Cloud          | Organization/Admin          |
| Permissions | Fixed set             | User-defined                |
| Updates     | Automatically updated | Managed manually            |
| Flexibility | Less flexible         | Highly flexible             |
| Use case    | Standard access       | Fine-grained access control |

---

## 4️⃣ Simple Example

Company policy:

* Developers can **start and stop VMs**
* Developers **cannot delete VMs**

Problem:
Predefined **Compute Admin** role allows deletion.

Solution:
Create a **Custom Role** with only required permissions.

---

✅ **Interview-friendly answer**

> Predefined roles are roles created and managed by Google that contain predefined sets of permissions for specific services. Custom roles are created by administrators where they can select specific permissions based on organizational requirements. Predefined roles are easy to use, while custom roles provide more control and follow the principle of least privilege.
