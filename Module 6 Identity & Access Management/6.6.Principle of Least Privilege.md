## Principle of Least Privilege (PoLP) in GCP IAM


## 1️⃣ What is the Principle of Least Privilege?

The **Principle of Least Privilege (PoLP)** means giving a **user, application, or service only the minimum permissions required to perform its job**.

Users should **not get extra permissions** that are not needed.

Simple definition:

> **Least Privilege means providing only the required access and nothing more.**

This is an important **security practice in cloud environments**.

---

## 2️⃣ Why Least Privilege is Important

It helps to:

* 🔐 Improve **security**
* 🚫 Prevent **accidental deletion of resources**
* 🛡 Reduce **risk of data leaks**
* ⚙️ Control **user access properly**

If users have **too many permissions**, they might accidentally or intentionally damage infrastructure.

---

# 3️⃣ Real-World Example (GCP DevOps Scenario)

### Scenario

A **developer** needs access to **view VM instances** for debugging.

### ❌ Bad Practice

Give the developer:

```
Compute Admin
```

This role allows:

* Create VM
* Delete VM
* Modify VM
* Restart VM

This is **too much access**.

---

### ✔ Good Practice (Least Privilege)

Give the developer:

```
Compute Viewer
```

Now the developer can:

✔ View VM details
✔ Check instance status

But cannot:

❌ Delete VM
❌ Create VM
❌ Modify VM

This follows the **Least Privilege principle**.

---

# 4️⃣ Another Real-World Example (CI/CD Pipeline)

### Scenario

A company uses **Terraform** to create infrastructure.

Terraform only needs to:

* Create VM instances
* Read network configuration

### Solution

Create a **custom role**:

```
compute.instances.create
compute.instances.get
compute.networks.get
```

Assign this role to the **Terraform service account**.

Now Terraform can **create infrastructure but cannot delete or modify other resources**.

# 5️⃣ Example in a Company Environment

| Team          | Required Access       | Role              |
| ------------- | --------------------- | ----------------- |
| Developer     | View logs             | Logging Viewer    |
| DevOps        | Manage infrastructure | Compute Admin     |
| Security Team | View audit logs       | Security Reviewer |

Each team gets **only the permissions needed for their work**.

---

# 6️⃣ Simple Interview Answer

> The Principle of Least Privilege means giving users or services only the minimum permissions required to perform their tasks. This improves security and prevents unauthorized or accidental changes to cloud resources. For example, if a developer only needs to view VM instances, they should be given the Compute Viewer role instead of Compute Admin.
