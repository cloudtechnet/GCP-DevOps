## GCP Global Infrastructure

**Google Cloud Platform (GCP)** runs on one of the **largest, fastest, and most secure global networks** in the world. Its infrastructure is designed to deliver **high availability, low latency, fault tolerance, and scalability** for applications used by millions of users globally.

A **Region** is a **specific geographical location** where Google operates data centers.

### Key Points:

* Each region represents a **country or large geographic area**
* Regions are **independent** of each other
* Used to meet **data residency, compliance, and latency requirements**
* Resources like VMs, databases, and services are deployed inside a region

### Why Regions Matter:

* Choose the nearest region to your users for **better performance**
* Use multiple regions for **disaster recovery (DR)**
* Helps comply with **government and industry regulations**

An application serving users in India is deployed in an India region to reduce latency.

## 2Ô∏è‚É£ Zones

A **Zone** is a **deployment area within a region**.

### Key Points:

* Each region has **multiple zones** (usually 2‚Äì4)
* A zone represents **one or more physical data centers**
* Zones are **isolated from each other** (power, networking, cooling)

### Why Zones Matter:

* Protect applications from **data center failures**
* Enable **high availability architectures**
* Applications can be deployed across multiple zones inside the same region

### Real-Time Usage:

* Run application servers in **multiple zones**
* If one zone fails, traffic is automatically served from another zone

## 3Ô∏è‚É£ Edge Locations

**Edge Locations** are **global points of presence (PoPs)** located close to end users.

### Key Points:

* Used mainly for **content delivery and network acceleration**
* Serve cached content like images, videos, and static files
* Reduce latency by serving data **closer to users**
* Connected to Google‚Äôs **private global fiber network**

### Why Edge Locations Matter:

* Faster page loads
* Better user experience
* Reduced load on backend servers

### Common Use Cases:

* Websites
* Streaming platforms
* APIs with global users

üìå Example:
A user in Europe accessing an application hosted in Asia receives static content from a nearby edge location instead of the main server.
