An **Organization** is the **top-level resource** in GCP.

### Key Points:

* Represents a **company or enterprise**
* Automatically created when a company uses **Google Workspace or Cloud Identity**
* Acts as the **root node** of the hierarchy
* Central place to apply **security policies and IAM roles**
* All folders and projects belong to an organization

### Why Organization Is Important:

* Enforces **company-wide policies**
* Centralized **identity and access management**
* Required for **enterprise-scale environments**

## 2ï¸âƒ£ Folder

A **Folder** is an **optional grouping layer** between Organization and Project.

### Key Points:

* Used to **group related projects**
* Can be **nested** (folders inside folders)
* Inherits policies from the Organization
* Helps structure resources by **department, team, environment, or application**

### Why Folders Matter:

* Simplifies **access control**
* Enables **policy inheritance**
* Makes large environments easier to manage

* By department: Finance, HR, IT

## 3ï¸âƒ£ Project

A **Project** is the **most important working unit** in GCP.

### Key Points:

* All GCP resources are **created inside a project**
* Each project has:

  * Project ID
  * Project Name
  * Billing account
* IAM roles are commonly assigned at the **project level**
* Acts as a **security and billing boundary**

### Why Projects Matter:

* Resource isolation
* Cost tracking and billing control
* Access control per application or environment

* One project for Dev environment
* One project for Prod environment
