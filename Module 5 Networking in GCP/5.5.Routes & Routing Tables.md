# Routes & Routing Tables (GCP)

In **Google Cloud Platform (GCP)**, **Routes** and **Routing Tables** control **how network traffic moves inside a VPC**.

In simple terms:

üëâ **Routes decide where traffic goes.**
üëâ **Routing table stores those rules.**

# 1Ô∏è‚É£ What Is a Route?

A **Route** defines:

* Destination IP range
* Next hop (where traffic should go)

Every route has:

| Component   | Meaning                         |
| ----------- | ------------------------------- |
| Destination | IP range (e.g., 0.0.0.0/0)      |
| Next Hop    | Internet gateway, VM, VPN, etc. |
| Priority    | Determines rule selection order |

---

# 2Ô∏è‚É£ What Is a Routing Table?

A **Routing Table** is a collection of routes.

* Every VPC has an automatic routing table.
* Routes apply to all subnets within the VPC.
* Subnets do not have separate routing tables in GCP.

---

# 3Ô∏è‚É£ Default Routes in GCP

When a VPC is created, GCP automatically creates:

### üîπ Default Internet Route


This allows VMs with external IPs to access the internet.

# 6Ô∏è‚É£ Next Hop Options

Routes can send traffic to:

* Default Internet Gateway
* VPN Tunnel
* Cloud Interconnect
* Internal VM (as router/firewall)
* Peered VPC network

---

# 7Ô∏è‚É£ How Routing Works (Simple Flow)

1. VM sends traffic to a destination.
2. GCP checks routing table.
3. Finds matching destination range.
4. Forwards traffic to the defined next hop.

---

# 8Ô∏è‚É£ Important Concepts

* Longest prefix match is used.
* More specific route (e.g., /24) overrides less specific route (/16).
* Lower priority value = higher priority.
* Routes are global in VPC.

---

# 9Ô∏è‚É£ Interview-Ready Example

If a route exists for:

* `10.0.0.0/16`
* `10.0.1.0/24`

Traffic to `10.0.1.5` will use `/24` route because it is more specific.
